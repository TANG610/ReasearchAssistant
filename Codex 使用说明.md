# Codex 使用说明

Codex 是一个可以在本地仓库里帮你读代码、改代码、运行命令、排查问题的 AI 编程助手。简单说，它适合做那些需要理解项目文件、执行小步修改、再验证结果的工作。

## 适合让 Codex 做什么

- 阅读项目结构，帮你解释某个模块或文件的作用。
- 修复报错、定位 bug，并尽量说明原因。
- 编写脚本、整理文档、补充配置。
- 按现有代码风格做小范围改动。
- 运行安全的检查命令，比如查看文件、搜索文本、执行测试。

## 推荐的协作方式

你可以直接描述目标，例如：

```text
帮我看一下这个项目怎么启动
帮我修复登录页的报错
帮我给这个脚本加一个参数
帮我整理一份 README
```

如果任务比较复杂，最好补充这些信息：

- 你希望改哪个功能。
- 现在遇到了什么现象或报错。
- 期望结果是什么。
- 有没有不能改的文件或限制条件。

## Codex 的默认工作流程

1. 先浏览项目结构，确认语言、框架和关键配置。
2. 再阅读和任务相关的文件。
3. 尽量用小改动解决问题。
4. 修改后运行可用的测试、构建或检查命令。
5. 最后说明改了什么、解决了什么、还有什么需要注意。

## 安全注意事项

- 不要把 API Key、Token、密码、Cookie、私钥直接发给 Codex 或写进代码。
- 修改登录、权限、支付、删除数据、删除文件相关逻辑时，要特别谨慎。
- 执行危险命令前要先确认风险。
- 如果发现仓库里已经有敏感信息泄露，应尽快更换对应密钥。
